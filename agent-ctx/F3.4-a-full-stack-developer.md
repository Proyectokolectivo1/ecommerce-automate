## Decisions
- Single-file client component (`'use client'`) per task instructions, instead of the server-component wrapper + view split used in `/dashboard/pedidos`. Reason: task explicitly asked for a client page.
- Three React Query keys: `['guides-stats']`, `['guides', status, carrier, debouncedSearch, page]`, `['guide', shipmentId]` (enabled when sheet open). `placeholderData: (prev) => prev` keeps previous page visible while fetching next.
- Custom `ShipmentStatusBadge` + `CarrierBadge` (the shared `StatusBadge` is bound to ORDER state-machine labels).
- Status palette per spec: CREATED=zinc, PRINTED=violet, IN_TRANSIT=teal, DELIVERED=emerald, RETURNED=rose. NO indigo/blue.
- `useMutation` for `POST /api/print?process=true` with `sonner` toast feedback + invalidation of both stats and list queries on success.
- PDF access gated: button + drawer download link shown only when `canAccess(role, 'BODEGA', 'SERVICIO')` (matches `/api/guides/[id]/pdf` RBAC).
- Print-queue button shown only when `canAccess(role, 'BODEGA')` (matches `/api/print` POST RBAC).
- Tracking timeline: vertical, most-recent first, dot + connector line per item, last item has no connector.
- Table sticky header, horizontal scroll on mobile (`overflow-x-auto`), max-height 640px with vertical scroll.
- Pagination footer identical pattern to orders page: prev/next + "Mostrando Xâ€“Y de Z" + page counter.
