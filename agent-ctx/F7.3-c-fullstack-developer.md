# F7.3-c — Documentación page

**Agent:** fullstack-developer (documentación subagent)
**Task:** Create `/dashboard/documentación` page with user + technical manuals using Tabs.

## What was built

Single client-component page at `src/app/(dashboard)/dashboard/documentacion/page.tsx` containing:

### Manual de Usuario (9 sections, in Cards)
1. **Introducción** — what the platform is, who it's for.
2. **Roles y permisos** — table with the 4 roles (ADMIN/GERENCIA/BODEGA/SERVICIO) + demo credentials.
3. **Navegación** — 4 sidebar sections (Operación/Logística/Análisis/Configuración) with the pages each contains.
4. **Gestión de pedidos** — FSM diagram (pre block) + individual transitions as Badge → Badge rows + actions per state.
5. **Flujo contra entrega (COD)** — 7-step ordered list + ASCII diagram.
6. **Dashboard y analítica** — how to read KPIs, finance period selector, charts.
7. **Inteligencia IA** — 4 generators (SALES_PREDICTION, ANOMALY, MONTHLY_SUMMARY, PRODUCT_ANALYSIS) with fallback note.
8. **Alertas** — table of 5 types (COD_UNPAID, GUIDE_ERROR, HIGH_RETURN, LOW_INVENTORY, SALES_DROP) + actions.
9. **FAQ** — 6 Q&A separated by `<Separator />`.

### Manual Técnico (11 sections, in Cards)
1. **Arquitectura** — hexagonal tree diagram + 3 mini-cards (Dominio/Adapters/HTTP).
2. **Stack tecnológico** — table (Next.js 16, TS 5, Prisma+SQLite, shadcn/ui, Recharts, TanStack Query, NextAuth, z-ai-web-dev-sdk, socket.io, in-memory cache/queue).
3. **Modelo de datos** — 17 Prisma models with one-line descriptions (grid 2 cols).
4. **FSM de pedidos** — 8 states + valid transitions diagram + Badge → Badge rows.
5. **Integraciones** — Shopify (webhook+Admin API+HMAC), Mastershop (dispatch+tracking), 5 pasarelas con PaymentProviderPort.
6. **Webhooks** — table of 3 receivers + idempotencia.
7. **Realtime** — bridge HTTP→WS, `io('/?XTransformPort=3003')`, 5 events, useRealtime hook.
8. **Seguridad** — NextAuth+JWT, RBAC, auditoría, sha256+salt demo with bcrypt/argon2 production note.
9. **Workers** — print-worker (15s) + alert-worker (5min) auto-started.
10. **Limitaciones conocidas** — 3 amber boxes (F4 reclassification, F5 in-memory aggregation, F6 SDK network dependency).
11. **Comandos** — table of `bun run dev/lint/db:push/db:generate/db:reset`, seeds, mini-service realtime.

## Helper components (defined in-file)
- `SectionHeading` — icon in rounded muted box + title + description, with accent color prop.
- `P`, `H3`, `UL`, `OL` — prose-like typography (muted-foreground body, list-disc/decimal markers).
- `Code` — inline `<code>` with bg-muted.
- `PreBlock` — `<pre>` with bg-muted/60, border, font-mono, overflow-x-auto.
- `BadgeTone` — Badge with 6 tones (emerald/amber/teal/violet/rose/zinc), no indigo/blue.
- `FSMRow` — `from → to` row using BadgeTone + ArrowRight.

## Color policy
NO indigo or blue. Used: emerald, amber, teal, violet, rose, zinc. Body text uses `text-muted-foreground`. Accent icons colored per section.
