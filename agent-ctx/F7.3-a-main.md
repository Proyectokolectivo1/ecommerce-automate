## UI/UX implementada
- **Access control**: `useSession` → skeleton durante loading; Card "Acceso denegado" (ShieldCheck amber) para no-ADMIN; queries con `enabled: isAdmin`.
- **Header**: "Gestión de Usuarios" + subtítulo + botón "Nuevo usuario".
- **KPI row (2→4)**: Total (Users), Activos (UserCheck emerald, borde emerald si >0), Inactivos (UserX zinc), Último login (Clock, relativeTime o "Nunca").
- **Filters bar**: search debounced 300ms + X clear, Select Rol, Select Estado, botón Limpiar, "Mostrando X–Y de Z", paginación Anterior/Siguiente.
- **Table (scroll-x móvil)**: Usuario (avatar circular con inicial+color de rol + nombre + badge "Tú" + email muted), Rol (badge amber/emerald/teal/zinc), Estado (badge emerald/zinc), Último login (relativeTime), Creado (dd/MM/yyyy, hidden móvil), Acciones (Editar + Eliminar rose, Eliminar disabled en propia fila, fila propia con highlight amber).
- **Create/Edit Dialog**: Email, Nombre, Contraseña (required create / opcional edit con hint), Rol (Select, lockRole para self), Activo (Switch solo edit). Validación inline. Loading en botón. key dinámico para remontar.
- **Delete AlertDialog**: "¿Eliminar usuario?" + email. preventDefault para no cerrar mientras muta. Botón rose.
- **Empty state**: Inbox + "No hay usuarios registrados" + Limpiar filtros.
- **Skeletons**: full-page (sesión), tabla (5 filas), campos dialog (4).
- **Toasts sonner**: crear/actualizar/eliminar success + error con mensaje backend.
- **Colores**: amber/emerald/teal/zinc (sin indigo/azul). Responsive. Footer sticky heredado.
